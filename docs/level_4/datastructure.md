# Level 4 Data Structure: ECS Component Schema

In ECS, "Tables" become "Components". For speed, historical time-series data is stored as columnar `.parquet` files inside `data/components/`, while DuckDB acts as the query engine linking them in-memory.

## 1. Base Entities
To optimize memory and join speeds, tickers are mapped to integer IDs during processing.
* `EntityMap:` `{ 1: "AAPL", 2: "MSFT", ... }`

## 2. Core Components (Parquet/DuckDB)
### `MarketDataComponent`
* `entity_id` (INT32), `date` (DATE32)
* `adj_close` (FLOAT32), `volume` (INT64)
* `daily_return` (FLOAT32) - *Pre-computed here to avoid the sparse-data pct_change teleportation bug!*

### `FundamentalComponent`
* `entity_id` (INT32), `filing_date` (DATE32)
* `revenue`, `total_debt`, `cash`, `shares_out` (FLOAT32)

### `MacroComponent`
* `date` (DATE32, PK)
* `vix`, `vix3m`, `tnx`, `spy` (FLOAT32)

## 3. Feature & Intent Components
### `FeatureComponent`
* `entity_id` (INT32), `date` (DATE32)
* `ev_sales_zscore`, `dynamic_discount_rate`, `dcf_npv_gap`, `beta_spy`, `beta_tnx`, `beta_vix` (FLOAT32)

### `ActionIntentComponent` (Replaces ml_predictions)
The raw intent generated by *any* strategy (XGBoost or Heuristic).
* `entity_id` (INT32), `date` (DATE32), `strategy_id` (STRING)
* `raw_weight` (FLOAT32)

### `TargetPortfolioComponent`
The final state after the Risk System modifies the ActionIntent.
* `entity_id`, `date`, `strategy_id`
* `target_weight`, `mcr` (FLOAT32)

## 4. Glass Box Audit Components (NEW)
### `RiskAuditComponent` (Parquet)
* `date` (DATE32)
* `covariance_matrix` (JSON/BLOB)
* `mcr_penalty_scalars` (JSON) - *Stores exactly how much each asset was penalized.*

### `XGBoostAuditComponent` (Parquet)
* `entity_id`, `date`
* `feature_vector` (JSON), `raw_prediction` (FLOAT32)
